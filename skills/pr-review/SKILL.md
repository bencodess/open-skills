# pr-review

Automated pull request review workflow. Analyzes diff quality, tests, coverage, and provides structured feedback with change requests.

## Prerequisites

GitHub CLI or access to PR URL

---

## How to Use

When reviewing a pull request:
1. Fetch the PR branch
2. Analyze diff for correctness, style, and edge cases
3. Check test coverage
4. Provide structured feedback with required/optional changes
