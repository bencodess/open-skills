# architecture

System architecture design and review. Evaluates trade-offs, proposes module boundaries, suggests patterns, and produces ADRs for decision tracking.

## Prerequisites

None

---

## How to Use

When designing or reviewing architecture:
1. Understand system constraints and requirements
2. Evaluate trade-offs between patterns
3. Propose module boundaries and data flow
4. Document decisions as ADRs
